# Difference between Stored Properties and Computed Properties in Swift

As a iOS developer, I often use Swift to write code. Swift is a powerful programming language that provides many useful features, including stored properties and computed properties. In this blog post, I will explain the difference between stored properties and computed properties in Swift.

Stored Properties
Stored properties are properties that store values directly in memory. When you define a stored property, Swift automatically creates a backing variable to store the property's value. You can access and modify the property's value using the dot syntax, just like you would access a variable.

Computed Properties
Computed properties are properties that calculate their value based on other properties or values. When you define a computed property, Swift doesn't create a backing variable to store the property's value. Instead, Swift calculates the property's value every time you access it.

In this example, we define a Circle class with a stored property radius and a computed property diameter. The diameter property calculates its value based on the radius property using the get and set accessors. When we access the diameter property, Swift calculates its value based on the radius property. When we set the diameter property, Swift calculates the new radius based on the diameter property and sets it.

Conclusion
Stored properties and computed properties are two important concepts in Swift. Stored properties store values directly in memory, while computed properties calculate their values based on other properties or values. By understanding the difference between stored properties and computed properties, you can write more efficient and elegant code in Swift.
